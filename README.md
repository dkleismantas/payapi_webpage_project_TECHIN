# Overview

I participated as a QA Tester in a 1.5‑month educational project at TECHIN, where our 6‑person team developed a fully responsive, accessibility‑friendly 4‑page website based on pre‑defined design documentation. The project followed Agile (Scrum) practices and simulated a real development environment, including task management, version control, and structured testing workflows.

## Technologies & Tools

**Languages:**
- HTML, CSS, JavaScript

**Development & Collaboration:**
- GitHub, Jira, Visual Studio Code

**Design Documentation:**
- Adobe XD, Figma

**Testing Tools:**
- Pixel Perfect Pro, WAVE Accessibility Tool, Microsoft Narrator, Chrome DevTools, Firefox DevTools

# Key Challenges

## Technical Challenges
- Ensuring WCAG‑aligned accessibility across all pages.
- Verifying pixel‑perfect UI implementation based on provided design files.
- Handling cross‑browser and cross‑device inconsistencies.

## Teamwork Challenges
- Aligning workflows among team members with different experience levels.
- Establishing clear communication rules and shared expectations.
- Adapting to Agile ceremonies and processes for the first time.

## What I Learned
- Applying Agile (Scrum) in a practical, team‑based environment.
- Conducting structured accessibility testing using industry tools.
- Collaborating effectively with developers and project stakeholders.
- Navigating real‑world QA workflows: task tracking, documentation, reporting, and regression cycles.
- Quickly adapting to new tools and integrating them into daily work.

# How to run the project
- Clone the repository.
- Open the project folder in Visual Studio Code.
- Install the Live Server extension in VS Code and click Go Live in the bottom‑right corner.
- Your browser should automatically open with the running project.

# Additional information
In the QA folder, you will find testing documents:
- a checklist that was used as the basis for performing the tests.
- In the Adobe folder, you will find the page design. Its images were uploaded to Pixel Perfect PRO to verify the page’s accuracy within a 5px tolerance.
